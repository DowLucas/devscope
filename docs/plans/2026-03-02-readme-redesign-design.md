# README Redesign Design

**Date**: 2026-03-02

## Problem

README reads like internal notes — leads with architecture, no visuals, mixes audiences, buries the getting-started flow.

## Design

Restructure README to lead with value prop + screenshot, separate self-hosting from contributor dev setup, move reference-level details lower.

### New structure

1. Title + badges (license, stars, Bun version)
2. One-paragraph pitch
3. Screenshot placeholder
4. Features bullet list
5. Getting Started: Prerequisites, Self-Hosting (Docker), Install the Plugin, Local Development
6. Architecture (brief diagram + package table)
7. Configuration (plugin + server env vars)
8. Contributing (short + link)
9. License (human-readable dual-license explanation)
10. Links (CONTRIBUTING, SECURITY, Issues)

### CONTRIBUTING.md

Minor tweaks to match README restructuring. Already solid.
